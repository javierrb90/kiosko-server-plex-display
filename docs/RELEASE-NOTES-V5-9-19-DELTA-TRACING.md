# v5.9.19 · Eventos delta y trazas detalladas

- Sustituye broadcasts globales en acciones de item por eventos delta:
  - `item:moved-to-deck`
  - `item:moved-to-backlog`
  - `item:completed`
  - `item:backlog-removed`
  - `item:deck-removed`
  - `item:completion-updated`
  - `item:completion-removed`
- El frontend actualiza Backlog, On Deck y Colecciones desde estado local sin recargar snapshot.
- Añade trazas detalladas:
  - `[action:start]`
  - `[action:end]`
  - `[action:error]`
  - `[delta]`
  - `[ws:broadcast]`
  - `[payload]`
  - `[snapshot]`
  - `[persist]`
  - `[slow-api]`
- WebSocket muestra activos al conectar/cerrar:
  - `[ws] conectado activos=N`
  - `[ws] cerrado activos=N`
- Snapshot/API pública sigue saneando `meta.raw` y objetos pesados.
- Persistencia write-behind/coalesced se mantiene.
- Variables útiles:
  - `TRACE_ITEMS=1` para trazas máximas de acciones;
  - `TRACE_WS=1` para todos los broadcasts;
  - `TRACE_PAYLOAD_BYTES=250000`;
  - `SLOW_API_LOG_MS=750`;
  - `PERSIST_DEBOUNCE_MS=350`.
